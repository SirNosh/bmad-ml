# Step 01 - Initialize

1. Define the research query or topic. Be precise -- vague queries produce survey-grade results, specific queries produce actionable intelligence.
2. Set scope parameters: breadth (survey across subfields) vs depth (deep dive into one technique). Document the choice and rationale.
3. Identify initial paper sources: arxiv preprints, Semantic Scholar, Google Scholar, domain-specific venues (NeurIPS, ICML, ACL, CVPR, etc.).
4. Define the taxonomy structure for organizing findings -- by method family, by application domain, by chronology, or hybrid.
5. Set a target paper count range (e.g., 15-30 for a focused review, 50+ for a broad survey) to calibrate effort.
6. Scan existing project artifacts for prior literature references that should be included or updated.
7. Check iteration history -- if this is a follow-up review, load the previous taxonomy and focus on delta papers since the last review date.
8. Confirm the research query, scope, sources, and taxonomy structure with the user before proceeding.

9. If operating in autonomous mode (invoked by Nosh), use the directive as the research query, accept default scope parameters, and skip the confirmation gate.

> Soft gate: "Here is the research scope and taxonomy plan. Anything to add or adjust before I begin the literature scan?"
