# Step 03 - Finalize

1. Produce the evaluation report using the template:
   - Executive summary: overall verdict (meets/does not meet quality bar) with key metrics highlighted
   - Detailed metrics table: all metrics, current version vs baseline, with confidence intervals
   - Failure analysis: categorized failures with counts, percentages, and representative examples
   - Human evaluation summary (if conducted): inter-annotator agreement, average scores per dimension
   - Methodology section: datasets, metrics, evaluation process, limitations

2. Highlight risks and recommendations:
   - **Critical risks**: Failure modes that could harm users, violate policies, or cause significant business impact
   - **Improvement opportunities**: Specific areas where targeted improvements could yield the highest metric gains
   - **Data recommendations**: Additional test cases or training data that would address identified weaknesses
   - **Re-evaluation triggers**: Conditions that should prompt the next evaluation cycle (model update, prompt change, usage pattern shift)

3. Present the evaluation report for review:
   - Full evaluation report document
   - Comparison summary (1-page overview for stakeholders)
   - Detailed failure case appendix
   - Recommendation priority list (highest impact improvements first)
   - Raw evaluation data location for further analysis
