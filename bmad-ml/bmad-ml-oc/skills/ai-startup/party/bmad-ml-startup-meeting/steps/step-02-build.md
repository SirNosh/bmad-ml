# Step 02 - Roundtable Discussion

1. Open the meeting by restating the agenda and format. The moderator (Nosh, acting as Startup CEO) sets expectations for focus, brevity, and decision quality.

2. **For each agenda item, run a focused roundtable - not parallel dispatch.** The moderator selects which agents are most relevant to this item and invites them to speak in turn. Each agent sees the full meeting transcript so far, including all prior agenda items' discussions.

3. The presenting agent leads with their update, finding, or proposal. The moderator then invites 2-3 other agents to respond based on relevance. Each responding agent must engage with what the presenter and prior responders said - agree, challenge, raise new concerns, or propose alternatives.

4. Each agent contributes from their specific expertise:
   - **Dumbledore** evaluates product fit, user value, and architectural coherence.
   - **Hermione** speaks to AI/ML engineering feasibility, model integration, and code correctness.
   - **Snape** raises safety, security, prompt injection, jailbreak, and abuse concerns - and will block a release if red lines are crossed.
   - **Luna** advocates for the end user, prompt quality, conversational UX, and output format clarity.
   - **McGonagall** assesses deployability, infrastructure cost, observability, and rollback readiness.
   - **Moody** scrutinizes evaluation rigor, test coverage, eval harness completeness, and regression risk.
   - **Hagrid** covers data freshness, ingestion pipeline health, schema stability, and data quality gates.

5. If a discussion drifts into pure research territory (experimental design, training methodology, novel architecture), the moderator should flag it and defer to `bmad-ml-all-hands` or `bmad-ml-research-party`.

6. The moderator steers the per-item discussion:
   - Redirects tangential threads (flag for parking lot or all-hands).
   - Ensures quieter agents with relevant expertise are invited to contribute.
   - Enforces time/turn limits per topic.
   - Drives toward a clear outcome: decision, question answered, action assigned, or deferral.

7. After each agenda item's roundtable concludes, the moderator states the outcome, records the decision with rationale and any dissenting views, and assigns action items with owners/deadlines/criteria. For Release Readiness meetings, explicitly record each gate-keeper's go/no-go vote.

8. Identify cross-cutting themes across agenda items. Flag these for strategic follow-up.

9. Reserve the final portion for parking-lot items raised during discussion. Decide whether each goes on the next meeting's agenda, escalates to `bmad-ml-all-hands`, or becomes a separate workflow.

10. Close with a round-table: each agent states their top priority or concern going forward.

> Hard gate: "All agenda items have been discussed. Confirm to produce the meeting notes, or add topics first." AI Startup workflows require explicit user approval at phase transitions.
