# Step 02 - Build and Analyze

1. For each paper, extract structured metadata: title, authors, year, venue, key contribution (one sentence), methodology summary, primary results, and stated limitations.
2. Build the taxonomy tree by assigning each paper to its category. Rebalance categories if any branch holds fewer than 2 or more than 10 papers.
3. Map citation relationships between papers -- identify foundational works (high in-degree) and recent extensions (high out-degree).
4. Identify research gaps: areas where the taxonomy has thin coverage, contradictory findings, or no recent work (>2 years stale).
5. Distinguish strong evidence (peer-reviewed, reproduced, large-scale) from preliminary findings (preprints, single-run, small datasets). Flag evidence strength per paper.
6. Note methodological trends: which techniques are gaining traction, which are being abandoned, and why.
7. For the top 5-10 most relevant papers, write a 2-3 sentence critical assessment: what they do well, where they fall short, and how they relate to the current project.
8. If any paper directly addresses the project's problem formulation, flag it as a "direct predecessor" and note how the current approach differs.
9. Cross-check that no major recent work (last 12 months) has been missed by searching for follow-up papers citing the foundational works.
10. Confirm the taxonomy, gaps, and key findings with the user before finalizing.

> Soft gate: "The literature taxonomy and gap analysis are ready. Anything to revise or any papers to add before I finalize?"
