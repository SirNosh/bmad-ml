# Step 03 - Finalize

1. Present the review decision with rationale:
   - Overall verdict: Approved / Changes Requested / Blocked
   - Summary of each review area with its individual verdict
   - Key strengths identified during the review
   - Key risks and their mitigation status

2. List required changes if not fully approved:
   - For each required change: description, severity (blocking vs non-blocking), owner, estimated effort, deadline
   - Define which changes must be completed before release vs which can be addressed post-release
   - Define the re-review process: full re-review vs targeted re-review of changed areas
   - Set a timeline for re-review

3. Present the full review report for discussion:
   - Review checklist with verdicts per area
   - Artifact completeness matrix (exists, current, complete for each artifact)
   - Quality gate status summary
   - Safety and evaluation findings summary
   - Required changes list (if applicable)
   - Release recommendation with conditions (if any)

**STOP and WAIT for user approval. Do not suggest the next workflow or chain to another agent. The user or Nosh will decide next steps.**
